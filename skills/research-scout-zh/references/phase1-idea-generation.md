# 阶段一：文献调研与想法生成

## 入口

四种输入模式，可组合使用：

```bash
# 模式一 —— 主题字符串
/research-scout-zh "电池 SOH 预测"

# 模式二 —— papers 标志（PDF 文件、论文名称或描述；可混用）
/research-scout-zh --papers paper1.pdf "Attention is All You Need" "transformer 2017 Google"

# 模式三 —— 组合模式（推荐）
/research-scout-zh "电池 SOH 预测" --papers paper1.pdf "BattFormer 2023"

# 模式四 —— 自由描述（可附带或不附带 PDF）
/research-scout-zh
# 用户随后输入一段描述自己想法的文字，并可附上 PDF
```

## 第一步 —— 解析输入

**模式一至三**：直接从命令参数中提取主题字符串和/或论文列表。

**模式四 —— 自由描述**：
提取以下结构，并在继续前展示给用户确认：

```
根据您的描述，我提取到以下内容：

**研究方向**：{方向}
**核心见解**：{见解}
**约束条件**：{列表}
**提供的论文**：
| # | 文件 / 描述 | 处理方式 |
|---|------------|---------|
| 1 | paper.pdf  | 直接读取 |
| 2 | {描述}     | 推断元数据 |

请使用 `/research-scout-zh confirm` 确认，或直接在线修改。
```

模式四确认后：自动将提取内容写入 `docs/user_requirements.md` 的阶段一章节，跳过手动填写步骤（第三步）。

## 第二步 —— 推断论文元数据（仅限名称/描述输入）

对每个非 PDF 论文条目，推断完整元数据并展示供确认：

```
我推断出以下论文信息，请确认：

| # | 完整标题 | 会议/期刊 | 年份 | 第一作者 | 一句话摘要 |
|---|---------|----------|------|---------|----------|
| 1 | Attention Is All You Need | NeurIPS | 2017 | Vaswani et al. | Transformer：纯注意力机制替代 RNN |

修正某条：/research-scout-zh revise-paper 1 "正确信息"
确认全部：/research-scout-zh confirm
```

`revise-paper` 仅修改指定行并重新显示完整表格。
PDF 输入跳过此步骤。

## 第三步 —— 读取用户需求

检查 `docs/user_requirements.md` → `## 阶段一` 章节。
- 若文件缺失或章节为空：使用 `references/user-requirements-template.md` 中的模板创建文件，告知用户填写阶段一章节，等待 `/research-scout-zh confirm`
- 若已填写（或由模式四自动填充）：读取并作为生成约束条件应用

## 第四步 —— 文献检索（双轨并行）

**轨道 A —— 以用户提供的论文为种子：**
- 直接读取 PDF：提取标题、作者、会议/期刊、摘要、方法章节
- 通过跟进参考文献列表和引用论文进行扩展

**轨道 B —— 自主 web_search：**
- 查询：主题 + 用户在 user_requirements 中声明的研究方向
- 覆盖范围：arXiv、NeurIPS、ICML、ICLR、CVPR、ECCV、ICCV、ACL、EMNLP、KDD、WWW
- 时间范围：近三年高引用论文 + 经典基础工作

合并并去重两条轨道的结果。目标：共计 ≥ 15 篇有效论文。

## 第五步 —— 检索自我反思循环

检索完成后，验证：每个声称的研究空白是否有 ≥ 2 篇论文作为证据？
若不满足，针对具体空白进行补充搜索。最多执行 3 轮。

## 第六步 —— 生成 3–5 个候选想法

从三个角度生成候选想法（每个角度至少一个）：

- **方法迁移**：将另一领域的成熟技术适配到本任务
- **现有方法改进**：修复当前最优方法的某个具体、已识别的缺陷
- **任务重构**：重新定义问题结构或引入新的先验知识

**每个候选的新颖性检验（必须执行）：**
搜索：`"{想法核心关键词}" site:arxiv.org OR site:openreview.net`
找到最相近的已有工作，标注相似度：高 / 中 / 低。
自动降级低新颖性候选 —— 不展示给用户。

**每个候选的自我批评（内部执行，结果写入卡片）：**
1. 声称的新颖性是真实的，还是仅仅是对现有方法的重命名？
2. 提出的实验是否真正验证了创新点（而非掩盖弱点）？
3. 基线选择是否公平（无挑选弱基线之嫌）？

**展示给用户：**

```
根据文献分析，以下是 {N} 个候选想法：

### 想法 1：{标题}
- **核心**：{一句话}
- **角度**：方法迁移 / 改进 / 重构
- **新颖性**：高 —— 最相近工作：{论文 [n]}，区别在于：{具体差异}
- **可行性**：中 —— 主要风险：{风险}
- **自我批评**：{内部审查结论，包含缺陷}

### 想法 2：...

使用 `/research-scout-zh pick {n}` 选择
```

用户执行 `/research-scout-zh pick {n}` 选定一个。

## 第七步 —— 深化所选想法

选定后：
- 扩展方法细节：模块拆解、数据流图（文本格式）、公式
- 可行性表格：计算资源、实施时间线、创新风险
- 基线选择及每项选择的理由

## 第八步 —— 引用内容核实

对草稿报告中每个 `>>` 标注：
1. 打开 `docs/papers/{标题}.pdf`
2. 在摘要 / 引言 / 方法章节中定位支撑句
3. 将核实文本附加到标注后：

```markdown
>> 该设计受 [3] 启发。[3]
>> 来源原文："We propose a selective state space model..." (第 3.1 节)
```

若 PDF 不可用：
```markdown
>> ... [3]
>> ⚠️ [低置信度：PDF 不可用，仅基于摘要]
```

若 PDF 可用但未找到支撑文本：
```markdown
>> ... [3]
>> ⚠️ [低置信度：在全文中未找到直接支撑文本]
```

将所有低置信度条目登记到 `## 待核实` 章节。

## 第九步 —— 写入 docs/idea_report.md

按照 `references/document-formats.md` 中的格式生成第一部分。

## 第十步 —— 下载 PDF

对参考文献列表中的每篇论文，按以下顺序尝试下载：
arXiv → Semantic Scholar → 作者主页 → OpenReview

保存路径：`docs/papers/{完整参考文献标题}.pdf`
去除字符：`/\:*?"<>|` —— 保留空格。

**若有论文下载失败**，在继续前提示用户：

```
以下论文无法自动下载：

| # | 标题 | 建议来源 |
|---|------|---------|
| 1 | {标题} | Sci-Hub / 作者邮件 / 图书馆 |

请将 PDF 放入 docs/papers/，文件名使用完整标题。
完成后执行 `/research-scout-zh confirm`，或执行 `/research-scout-zh skip-papers` 跳过。
```

收到 `confirm`：重新扫描 `docs/papers/`，将新文件与参考文献条目匹配，继续执行。
收到 `skip-papers`：将剩余论文在参考文献列表中标记为 `[PDF 不可用]` 并继续。

用户通过 `--papers` 提供的 PDF：复制到 `docs/papers/`，重命名为完整标题。

## 第十一步 —— 阶段一审核检查点

展示 PDF 摘要，然后停止：

```
PDF 摘要：
- ✅ {标题}
- ✅ {标题}（用户上传）
- ❌ {标题} [PDF 不可用]

docs/idea_report.md 已生成，请审阅，然后：
- /research-scout-zh step2   —— 确认并进入实验设计
- /research-scout-zh revise "反馈"  —— 根据修改意见重新生成第一部分
```

在用户执行 step2 或 revise 之前，不得继续推进。
