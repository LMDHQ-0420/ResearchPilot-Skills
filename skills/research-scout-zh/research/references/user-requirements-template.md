# user_requirements.md 模板

本文件由 Claude 通过对话收集信息后自动创建和维护。**用户无需手动编辑此文件。**

Claude 在每个阶段开始前，若用户输入中信息不足，会在对话中主动提问收集，收集完成后将结果写入本文件对应章节，作为后续执行的约束依据。

**规则：**
- 本文件由 Claude 写入和维护，不由用户直接编辑
- Claude 绝不将本文件中的内容复制到 `idea_report.md` 或 `dev_log.md` 中
- 主文档只展示执行结果；本文件保存收集到的用户需求原文
- 用户可在对话中随时声明文档偏好（语言、详细程度等），Claude 会更新本文件对应字段

---

## 文件格式

```markdown
# User Requirements — {Topic}
> Created: {YYYY-MM-DD} | Project: {project_name}
> 由 Claude 根据对话内容自动填写，用户无需手动编辑。

---

## 阶段 A：方向偏好
> Claude 在阶段 A 开始时通过对话收集，用于指导候选 idea 方向的生成。

### 希望侧重的方向
{从用户对话中提取的偏好方向，如轻量化/可解释性/跨域泛化等}

### 不希望涉及的方向
{从用户对话中提取的排除方向}

### 其他约束
{从用户对话中提取的其他方向限制}

### 文档偏好
{从用户对话中提取的文档语言、详细程度等偏好，默认：中文正文 + 英文章节标题}

---

## 阶段 C：硬件约束
> 仅记录对实验方案有硬性影响的约束，不记录超参数偏好。

### GPU 约束
{从用户对话中提取的 GPU 数量上限、显存上限；例如"最多单卡 A100 80G"}

### 训练时长上限
{从用户对话中提取的单次训练时长限制，若无限制则留空}

---

## 阶段 D：实现约束
> 仅记录对代码结构有决定性影响的约束。

### 基于已有项目
{从用户对话中提取的已有项目路径或链接，或"从头开始"}

### 框架偏好
{从用户对话中提取，默认 PyTorch}

### 其他硬性要求
{从用户对话中提取的硬性编码要求，如"必须支持多卡"、"需要 ONNX 导出"；偏好性要求不记录}
```

## Claude 收集需求的时机

### 阶段 A（方向探索）
若用户输入中未包含足够信息，Claude 在对话中一次性提问：

```
在开始检索前，了解几点（可跳过任意一项）：
1. 你希望从哪个角度切入？（如：轻量化、可解释性、跨域泛化）
2. 有没有不希望涉及的方向？
3. 有其他约束吗？
```

若用户输入已足够详细，直接提取并写入，不再提问。

### 阶段 C（实验设计）
在开始设计实验前，Claude 在对话中提问：

```
在设计实验前，确认两点：
1. GPU 约束？（数量上限 + 显存，如"最多单卡 A100 80G"）
2. 单次训练有时间限制吗？
```

### 阶段 D（实现设计）
在开始生成 implementation.md 前，Claude 在对话中提问：

```
在设计实现方案前，确认几点：
1. 框架用 PyTorch 还是其他？
2. 有没有基于已有开源项目？（如有，提供链接）
3. 有硬性要求吗？（如：支持多卡 DDP、需要 ONNX 导出）
```
